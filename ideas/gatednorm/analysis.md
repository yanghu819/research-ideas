# GatedNorm 深度分析

## 1. 机制拆解

GatedNorm 定义为：

\[
y=\mathrm{RMSNorm}(x),\quad y_g=\sigma\left(W_{up}(\mathrm{swish}(W_{down}(y)))\right),\quad y'=y_g\odot y
\]

本质是：在 RMSNorm 后添加低秩逐维门控，显式提供“动态重缩放”能力。

## 2. 理论直觉（基于论文论证）

2601 工作提出 outlier-driven rescaling 视角：

1. attention sink 与 residual sink 都可看作“借归一化做缩放”的现象。
2. 当显式门控存在时，模型对异常值的依赖可降低。
3. 激活更平滑，量化更友好。

## 3. 主要收益

1. 训练性能提升：论文报告平均分提升趋势（不同规模有收益）。
2. 量化鲁棒性增强：低比特量化掉点更可控。
3. 架构鲁棒性提升：对会抑制 outlier 的激活/结构更不敏感。

## 4. 成本与工程代价

1. 参数增加（2B示例约2%）。
2. 延迟增加（小模型更明显，规模增大相对影响变小）。
3. 需要精细调参：rank、激活函数、层位选择。

## 5. 风险与失败模式

1. 全层直接加 gate 导致成本偏高。
2. 使用不合适激活（无界或不稳定形态）可能导致训练不稳。
3. 与其他动态scale机制并用时，出现功能重叠和收益递减。

## 6. 实施建议

1. 层位：优先后1/3层或FFN前Norm。
2. 门控：优先 elementwise + sigmoid。
3. rank：从小到大扫（如 8, 16, 32）。
4. 观察：
- outlier 指标（sink比例、激活峰值）
- 验证损失
- W8A8/W4A4 掉点
- 端到端延迟

## 7. 何时优先使用

1. 你已经观察到 residual sink/outlier 尾部很重。
2. 你准备上线低比特量化，且当前掉点大。
3. 你接受少量推理开销换稳定收益。
